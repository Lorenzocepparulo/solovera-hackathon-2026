## Context
The user wants to optimize the landing page based on PageSpeed Insights (or similar tool) feedback. A comprehensive codebase audit identified multiple performance issues across HIGH, MEDIUM, and LOW severity.

## Phase 1: HIGH Severity Fixes

### 1.1 Remove stale root `index.html`
- The root `index.html` is a manually maintained all-in-one version with inlined CSS/JS and base64 images. It duplicates `src/index.html`.
- **Action:** Delete or rename `index.html` from project root to prevent accidental serving.

### 1.2 Fix Twitter image domain mismatch + typo
- **File:** `src/index.html` line 19
- Current: `twitter:image` uses `https://hackathon.solovera.it/og.png`
- Should be: `https://hackathon.solovera.work/og.png` (matching og:image URL)
- Also fix typo: `hackaton` → `hackathon` in any meta tags if present.

### 1.3 Optimize jury member images
- **Files:** `src/index.html` lines 134, 139, 144
- Add `width`, `height`, `loading="lazy"`, and `decoding="async"` attributes to `<img>` tags for LCP/CLS improvement.
- Consider converting JPGs to WebP format (smaller file size, same quality).

## Phase 2: MEDIUM Severity Fixes

### 2.1 Optimize canvas star animation
- **File:** `src/modules/stars.ts`
- **Changes:**
  - Reduce star count from 200 to ~80 (still visually rich, much less CPU)
  - Add `document.hidden` check — pause animation when tab is not visible
  - Debounce the resize handler (use 150ms like sticky-cta.ts)
  - Handle `devicePixelRatio` for sharp rendering on Retina displays

### 2.2 Optimize comet particles
- **File:** `src/modules/comets.ts`
- **Changes:**
  - Check `window.matchMedia('(prefers-reduced-motion: reduce)')` before starting setInterval
  - Consider reusing a single DOM element instead of create/destroy cycle
  - Add visibility check to skip spawning when tab is hidden

### 2.3 Throttle ship scroll handler
- **File:** `src/modules/ship.ts`
- **Change:** Apply the same 150ms `Date.now()` throttle pattern from `sticky-cta.ts` to the scroll handler (lines 8-14). `getBoundingClientRect()` on every scroll event causes layout thrashing.

### 2.4 Self-host Google Fonts (optional but impactful)
- **File:** `src/index.html` line 22
- **Current:** 3 font families (Unbounded, Space Grotesk, Space Mono) loaded from `fonts.googleapis.com` — render-blocking external request.
- **Option A:** Self-host fonts by downloading WOFF2 files to `public/fonts/` and using `@font-face` in CSS. Eliminates DNS+TLS round trips to Google.
- **Option B:** Keep Google Fonts but add `font-display: optional` to avoid FOUT entirely (trade-off: users may not see custom fonts on slow connections).
- **Recommendation:** Option A for best performance.
